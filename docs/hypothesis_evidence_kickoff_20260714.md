1. **매진 → 매출 무영향**: 개별 품목 매진은 카테고리 내 수요 흡수로 대체되어 매출 손실로 이어지지 않음. 4매장 중 3곳(광교·광화문·삼성타운) 무영향, 메세나폴리스만 약하고 강건하지 않은 손실 신호.

3. **매장별 분리 학습 우수 (모델이 아니라 학습 데이터를 분리)**: 동일 모델을 4매장 통합 학습하면 전 매장 WAPE 1.2~2.9pp 악화 — 매장 특성이 구조적으로 달라 신호가 희석됨. 단, 하이퍼파라미터 세밀 튜닝은 과적합 → "학습 데이터는 분리, 설정은 통일". 향후 매장 확대 시 유사 매장 군집 단위 합산 학습 가능.

## H1. 매진 → 매출 영향: 개별 품목 매진은 매출 손실 없음 (수요 흡수)

### 주장
품절이 나도 고객은 같은 카테고리의 다른 품목으로 옮겨가므로(수요 흡수), **개별 품목의 매진이 곧 매출 손실은 아니다**. 매출은 카테고리 총수요 수준에서 결정된다.

### 1차 근거 — 메커니즘 (affirmative evidence)
"매진이 손실이 아니다"의 직접 근거는 수요가 어디로 가는지를 보는 것:

- **영수증 단위 substitution DiD β ≈ 0**: 한 품목 품절 전후로 대체 품목 판매가 유의하게 늘지도/줄지도 않음 — 개별 substitution 효과는 약하되, 카테고리 총량은 보존.
- **MNL / Nested-Logit λ ≈ 0.99**: 광교에서 카테고리는 사실상 "한 묶음 demand"로 행동. 품목 간 전환 비용이 낮아 매진이 총수요를 깎지 않음.
- 즉 **진짜 리스크는 개별 품목 매진이 아니라 카테고리 공급 부족**. 카테고리 총량만 맞으면 개별 매진은 매출 중립.

### 2차 근거 — 정합성 확인 (회귀)
매진→매출 회귀로 위 메커니즘과 모순되지 않는지 교차 확인. 매진은 수요에 내생적(수요↑ → 매출↑ + 매진↑)이라 단순 상관은 무의미 → **트래픽(영수증 수)·달력 통제 후** early_share(15시 이전 매진 비율)의 한계효과를 측정.

**2026-07-10 재검증 (stockout 정의 교정 반영, `verify_stockout_revenue_fixed.py`)**:

| 매장 | early_share 한계효과 | p-value | 판정 |
|---|---|---|---|
| 광교 | **+2.74%** | 0.196 | 무영향 (부호는 오히려 수요신호) |
| 광화문 | +1.78% | 0.358 | 무영향 |
| 삼성타운 | −1.77% | 0.416 | 무영향 |
| 메세나폴리스 | −4.86% | 0.021 | 약한 손실 신호 (비강건) |

- 광교·광화문 계수가 **양(+)** — 손실이 검출되기는커녕, 일찍 매진되는 날일수록 매출이 높음(고수요일의 내생성 잔존). 수요 흡수 가정과 일관.
- **삼성타운**: 이전 분석의 −8.3% "강건한 lost-sales 신호"는 **stockout_time 버그**(로더가 하루 다중 품절 이벤트 중 첫 것만 취해 매진 시각이 인위적으로 앞당겨짐)의 아티팩트였음. 교정 후 신호 소멸(−1.77%, p=0.416). → 삼성타운 특별대우 근거 소멸.
- **메세나폴리스**: 유일한 유의 신호이나 교차검정(매진 중앙시각 변수, L4b)에서 강건성 미확증(p=0.232). 4매장 × 2검정 중 1건 유의는 **다중비교상 우연 범위**. "약신호이나 강건하지 않음."

### 한계 (반드시 함께)
- **무매진일이 거의 0** (광교 0 / 삼성 0 / 메세나 1 / 광화문 4일) → "매진이 전혀 없었다면" counterfactual을 식별할 수 없음. 측정값은 총손실이 아니라 *이른 매진 vs 늦은 매진의 한계효과*.
- 전부 **associational** (early_share가 내생). 진짜 walk-away 손실(못 산 고객)은 데이터에 기록되지 않음.
- 교정 후 매진이 대부분 저녁(중앙 19시)이라 15시 이전 "이른 매진"이 희소 → 검정력 자체가 낮음(대부분 p>0.2). "무영향"에는 "이 임계값으론 식별력 부족"도 섞여 있음.

### 결론
메커니즘(수요 흡수)이 1차 근거, 회귀는 정합성 확인. **개별 품목 매진 = 매출 손실 없음**이 3/4 매장에서 일관되게 지지되고, 유일한 예외(메세나)도 강건하지 않음. → 모델 target `adjusted_demand`(검열된 매진일 판매량 보정)와 카테고리 합 예측 접근이 정당화됨.

## H2. 폐기·마감 손실 규모: 광교 단독 연 3,924만원 (원가율 0.30 가정)

### 정정 (초기 9,102만원 → 3,924만원)
초기(2026-05-26) 산정 시 **아띠제로부터 원가율을 제공받지 못해**, 폐기 비용을 `폐기수량 × 판매단가(UM_SO)` — 즉 **버린 빵을 정가 전액에 판 값**으로 계산했다(사실상 원가율 100% 가정). 이는 ① 어차피 팔리지 않았을 수요를 정가로 세고 ② 매진 손실과 이중계상되므로 폐기 손실을 과대평가한다. 폐기의 실제 실손은 **버린 원재료비(원가 기준)**이므로, 현재 표준(원가율 0.30, `business_metrics.py`)을 적용해 재계산한다.

### 데이터 신뢰도 — 폐기수량은 재고 실측
재고정보(QT_MADE, QT_OUT)로 항등식 검증: `생산(made) − 판매(정상+마감) − 폐기(out) ≈ 0`이 매장별 **90~93% 정확 일치**. → 폐기수량 QT_OUT은 capacity 추정이 아니라 **재고 실측값**. 신뢰 가능.

### 재계산 결과 (원가율 0.30)

| 매장 | 폐기수량(5년) | 폐기 원가(30%, 5년) | 마감 할인 손실(실측, 5년) | **연환산 총손실** |
|---|---|---|---|---|
| **광교** | 75,950 | 1.11억 | 8,519만 | **3,924만/년** |
| 광화문(3.5년) | 53,640 | 0.77억 | 4,658만 | 3,577만/년 |
| 메세나폴리스 | 69,873 | 0.98억 | 4,714만 | 2,904만/년 |
| 삼성타운 | 69,249 | 0.99억 | 2,534만 | 2,483만/년 |

**광교 연 3,924만원** = 폐기 원가 2,220만 + 마감 할인 1,704만.

### 원가율 민감도 (광교, 연환산)
폐기 손실은 원가율에 **선형 비례**. 마감 손실(1,704만)은 실제 지급된 할인액(AM_DC)이라 원가율과 무관.

| 원가율 가정 | 폐기(연) | 마감(연) | 합계(연) |
|---|---|---|---|
| 1.0 (판매가 전액 — 초기 산정) | 7,398만 | 1,704만 | 9,102만 |
| 0.5 | 3,699만 | 1,704만 | 5,403만 |
| **0.3 (표준, 채택)** | 2,220만 | 1,704만 | **3,924만** |
| 0.2 (발주 역산 추정) | 1,480만 | 1,704만 | 3,184만 |

### 한계
- **원가율 0.30은 가정** — 실측 미수령(데이터 요청 항목 #7). 발주 패턴 역산으로 빵 ~0.20 / 페이스트리 ~0.28 추정. 원가율 실측 수령 시 폐기 손실 부분은 선형으로 갱신.
- 마감 할인 코드는 closing 라벨(0077 30%, 0069 20%)만 폐기 proxy로 사용. 92.8%가 20시 이후 발생.
- 폐기수량은 실측이나, 폐기 원인(과잉생산 vs 수요붕괴)은 별도 분석 필요.

## H3. 통합 vs 매장별 분리 모델: 분리가 우수

### 결론 (구조적)
4매장을 하나로 합쳐(pooling) 학습하면 **전 매장 WAPE가 악화**:

| 매장 | 단독 학습 WAPE | 통합 학습 WAPE | Δ |
|---|---|---|---|
| 광교 | 13.95% | 15.35% | **+1.40pp** |
| 광화문 | 15.32% | 18.21% | **+2.89pp** |
| 메세나폴리스 | 13.95% | 15.10% | **+1.16pp** |
| 삼성타운 | 21.91% | 23.94% | **+2.03pp** |

원인: 매장 특성이 **구조적으로 다름** — 광교는 주말 강세(토 368), 삼성타운은 주말 절벽(토 126/일 100). 매진 중앙시각도 삼성 13:31 ~ 메세나 17:16으로 상이. 카테고리 간 상관(bread-pastry)도 매장별 0.15~0.40으로 제각각. 통합 학습은 이 차이를 평균화로 희석시킴. 매장별 유효 피처도 **합집합 33개 / 교집합 1개(lag14만 공통)**.

### 핵심 — "다른 모델"이 아니라 "다른 학습"
여기서 '분리'는 매장마다 **다른 모델(알고리즘·구조)을 쓰는 게 아니다**. **동일한 모델·피처·하이퍼파라미터를 매장별 데이터로 따로 학습**하는 것 — 즉 바꾸는 것은 모델이 아니라 **학습 데이터의 범위(scope)**다.
- 통합이 나쁜 이유도 "모델이 부족해서"가 아니라, 서로 다른 수요 구조를 **한 파라미터 세트로 평균내면서 각 매장 신호가 희석**되기 때문.
- 따라서 성능 레버는 모델 아키텍처가 아니라 **학습 데이터를 어떻게 묶느냐**에 있다.

### 뉘앙스 (정직하게) — 학습 scope는 분리, 설정은 통일
"분리가 낫다"가 **매장별 하이퍼파라미터 세밀 튜닝**까지 정당화하지는 않는다. 매장마다 학습 window·피처 수를 backtest 최적으로 고르는 방식(customization)은 **16개 목요일 backtest noise에 과적합**:
- Hold-out 검증: 앞 8주 최적 조합이 뒤 8주 최적과 **4매장 모두 불일치(0/4)**.
- Bootstrap: 광교에서 (window, N=4)와 (window, N=33)이 동률 선택 — 정반대 설정이 같은 빈도.

→ 권장 운영: **학습 데이터는 매장별로 분리, 모델 설정(피처·하이퍼파라미터)은 통일**(예: 전 매장 5Y window + N 앙상블). 분리의 이득은 취하고 튜닝 과적합은 피함.

### 확장 전망 — 유사 매장 군집화 시 합산 학습 가능
현재 4매장은 특성이 너무 달라 통합(pooling)이 손해다. 그러나 이는 "매장별 완전 분리가 최종형"이라는 뜻이 **아니라**, 현재 표본(4매장, 각기 다른 상권)에서의 최적해일 뿐이다.
- **매장 범위를 확대해 충분한 표본이 모이면, 유사 매장끼리 군집화(clustering)** 하여 **군집 단위 합산 학습**이 가능할 것으로 판단한다.
- 군집 내에서는 수요 구조가 비슷해, pooling의 이점(표본 수↑, 신규·저데이터 매장 cold-start 대응, 안정성)이 희석 손해를 능가할 수 있다.
- 지향점은 계층적 구조: **군집 단위 합산 학습 → 매장 단위 fine-tune**. 신규 매장·소규모 매장으로 확장할수록 이 방향의 가치가 커진다.

### 각주 (수치 기준일)
위 backtest의 target은 **당시에도 `adjusted_demand`** (= `sold_normal + α × sold_closing`, α=0.5 → 이후 0.7)로, **수요 흡수 가정을 이미 반영**한 값이다("흡수 미반영 옛 모델"이 아님). adjusted_demand는 마감 *할인* 기반이라 이후의 stockout 라벨 재정의에도 영향받지 않는다. 절대 WAPE는 **2026-05-26~30 측정치**로 이후 α·bulk 필터 등 세부 세팅이 일부 갱신됐으나(target 개념은 동일), **상대 결론(통합 < 분리)은 구조적이라 유지**된다.
