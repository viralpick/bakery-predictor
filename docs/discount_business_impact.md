# Discount 데이터 기반 비지니스 임팩트 분석

광교 1매장 5년치 (2021-01-01 ~ 2025-12-31), 보나비 xlsx 5시트.
폐기 실측 데이터 도착 전, **할인 데이터로 정량화한 비지니스 임팩트** 정리.

---

## TL;DR

비지니스 가치는 2축으로 나뉜다.

### ① 마감 할인 → 폐기 최소화 (정량 가능)
- 현재 광교는 **연 13,355 unit (≈ 일 36 unit)** 을 마감 시간대 28% 할인으로 회수 중
- 직접 측정 매출 손실 **연 1,569만원** (정가 - 할인가)
- 이상적(발주 적정화)으로 마감 할인 50% 줄이면 **연 ~800만원 추가 매출 회수**
- 마감 할인을 100% 없애려면 일 36 unit 감산 필요 (단 인기 품목까지 줄이면 매출↓ 위험)

### ② 인기도 기반 수량 조정 (107개 품목 → 5단계 권장)

| 구간 | 품목 수 | 핵심 |
|---|---|---|
| Strong Up | 11 | 평균 13시 이전 품절 + 마감에 거의 안 남음 → **수량 적극 증가** |
| Up | 24 | 일찍 품절 OR 마감 적음 OR 추세 +15%↑ |
| Hold | 34 | 안정 |
| Down | 24 | 늦게 품절 OR 마감 많음 OR 추세 -15%↓ |
| Strong Down | 14 | 늦게 품절(17시+) + 마감 할인 14%+ → **수량 적극 감소** |

대표 사례:
- **Strong Up**: 올리브치아바타 (10.9시 품절, 추세 +28%), AOP 버터 맘모스빵 (8.5시 품절)
- **Strong Down**: 버터모닝롤 (17.8시 품절, 마감 할인 18%, 5년 4,593,140원 손실 1위), 식빵의정석 (마감 할인 21%, 추세 -30%)

## §2. 인기도 기반 수량 조정 (107개 품목)

### 신호 4종 통합

광교는 거의 모든 품목이 매일 품절 (stockout_freq 평균 94%) → 절대 기준 안 통함. **per-category 분위수 기반**으로 변환:

1. **avg_stockout_h**: 카테고리 내 하위 25%(일찍) / 상위 25%(늦게)
2. **closing_rate_per_sold**: closing_qty / total_sold — 하위/상위 25%
3. **trend_pct**: 최근 90일 vs 직전 90일 평균 매출 변화 (±15% threshold)

각 신호 +2/-2/+1/-1 점수 합산, 5단계 분류.

### 추정 폐기량 (신뢰도 ★★ — 입고량 부재)

```
estimated_waste_qty = capacity − sold_units − closing_discount_qty
```

`capacity`는 `bonavi_daily.parquet`에서 자체 추정한 95-percentile 기반 값이라 **과대 추정 위험**. 산출만 해두고, 실제 폐기 수량 데이터 도착 시 `WasteEstimator` Protocol을 swap해서 갱신.

| 카테고리 | 총 capacity | sold | closing | est_waste | 추정폐기율 |
|---|---|---|---|---|---|
| bread | 421,701 | 210,820 | 27,867 | 184,842 | 43.8% |
| pastry | 650,135 | 255,331 | 32,798 | 363,868 | 56.0% |

| sweets | 34,589 | 12,488 | 1,780 | 20,402 | 59.0% |

폐기율 50% 안팎은 capacity 추정 보수성 탓 — 실측 도착 시 재계산 필수.

## §4. 시나리오 손익 비교 (cost rate 30%, 마진 50% 가정)

3가지 발주 전략 비교:

| 시나리오 | 마감 할인 회수 | 원가 손실 | 정가 매출 손실 | 순 손익 |
|---|---|---|---|---|
| **A**: 마감 할인 없이 폐기 | 0 | 1,664만원 | — | **-1,664만원** |
| **B**: 현재 (마감 할인으로 회수) | 3,975만원 | 1,664만원 | 1,569만원 | **+2,312만원** |
| **C**: 적정 발주 + 정가 100% 판매 (이상) | — | 0 | 0 | **+2,773만원** |

**B → C 임팩트: 연 461만원** (현재 마감 할인 매출 손실분이 보전됨)

⚠️ C는 이상치. 현실 시나리오는 "마감 할인 50% 줄이기" → **연 ~800만원 매출 회수** + 폐기 비용 절감 추가.

### v2_q85 production model이 노릴 목표

- 마감 할인 50% 감소 (현재 일 36 unit → 일 18 unit) = **연 785만원 직접 매출 회수**
- 단, 인기 품목까지 줄이면 안 됨 → §2 strong_up 11개는 발주 늘리고, strong_down 14개는 줄이는 차별 발주
- 인기 품목 공급 확대 시 추가 가치 = 정량화 어려우나 고객 만족도/리텐션 ↑

## §5. 할인 데이터 한계 + 다음 단계

### 한계

1. **폐기 실측 부재**: 마감 할인은 폐기 **lower bound**. 마감 할인도 못 받은 잔량은 측정 불가. 폐기 데이터 도착 시 `WasteEstimator` Protocol swap.
2. **단일 매장**: 광교는 "진짜 무품절일 0일" 매장 (baseline 부재). 추세 분석은 OK이나 인과 추정은 다매장 데이터 필요.
3. **PAYCO 30% (0121) 큰 비중**: 결제수단 할인이라기엔 비율이 크다. 마감 시간 비중 2.3%이니 마감과 무관 확실하나, "PAYCO 자체가 마케팅 이벤트성 50% 할인"일 가능성. PM 확인 필요.
4. **set/단품 구분 미반영**: 셋트 상품에서 할인이 어떻게 잡히는지 검증 필요.

### 다음 단계 (우선순위)

| # | 작업 | 입력 | 산출 |
|---|---|---|---|
| 1 | 할인 row 분리 → `potential_demand` 재추정 | 본 분석 | 모델 정확도 |
| 2 | 영업시간 + 휴무일 → `open/close_hour` dynamic | PM 제공 | 마감 시간대 노이즈 제거 |
| 3 | **폐기 실측 swap** | PM 제공 | 추정 → 실측, ROI 재계산 |
| 4 | 다매장 데이터로 인기도 분석 검증 | PM 제공 | 광교 특유 패턴 vs 보편 패턴 식별 |
| 5 | 영수증 패턴 검증 — "카테고리 = 한 묶음" 가설 | 본 데이터 | framework_pivot 강도 |

## §6. 모듈 + 인터페이스

신규 분석 모듈 3종 (`src/bakery/analysis/`):
- `discount.py` — 할인 데이터 로드 + 30개 코드 분류 + 마감 할인 추출
- `popularity.py` — 4신호 통합 + 분위수 기반 권장
- `waste.py` — 폐기 추정 (실측 swap 가능한 `WasteEstimator` Protocol) + 마감 할인 손실 계산

핵심 인터페이스:
```python
class WasteEstimator(Protocol):
    def per_day_item(self) -> pd.DataFrame:
        """date, item_id, waste_qty, waste_cost_won"""
```

실측 데이터 도착 시 `ActualWasteFromIngestion(ingestion_df)` 같은 새 클래스만 구현, 위 모든 분석 그대로 작동.

## §7. 비지니스 결정 액션 (PM 전달용)

### 즉시 실행 가능 (데이터 충분)

1. **Strong Up 11개 품목 발주량 +20%** → 일찍 품절되는 인기 품목 공급 확대
2. **Strong Down 14개 품목 발주량 -20%** → 마감 할인 직접 감소
3. 식빵류(C-prefix PB) 라인 발주 전면 재검토 — strong_down 비중 큼

### 측정 KPI (90일)

- 마감 할인 row 수 (목표: 50% 감소)
- 일평균 마감 할인 매출 손실 (현재 43,000원 → 목표 21,500원)
- Strong Up 품목 평균 품절 시각 (현재 11시 → 13시 목표)
- 카테고리별 wMAPE / Stockout hours / Waste rate (PM 평가 framework)

### 데이터 받으면 즉시 (1순위 폐기)

폐기 실측 + 입고량 도착 시:
- 추정 폐기율 50% → 실측으로 검증/갱신
- v2_q85 발주 정확도 vs 실제 폐기 ROI 재계산
- `business_metrics.py` cost 계산 추정 → 실측 교체
