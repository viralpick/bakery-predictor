# v4 Framework vs Naive Baseline + 추가 데이터 필요성

**광교 매장 PoC 정직 평가**: 단순 dow/월 평균 baseline 대비 v4 framework의 진짜 부가가치 + 부족한 데이터 + 데이터 보충 시 기대 효과.

---

## §1. 정량 비교 — Naive Baseline vs v4 Ensemble

운영 호환 backtest (D=목요일 → D+4~D+10, 16 Thursdays × 7 horizons).

### 1-1. 핵심 metric

| 모델 | WAPE | 매진율 | 평균 발주 | 발주 over |
|---|---|---|---|---|
| **Naive (target_dow 최근 4주 평균)** | **18.18%** | **45.5%** | 247 unit | +5 |
| Naive (target_dow 최근 12주 평균) | 18.45% | 35.2% | — | — |
| **v4 Ensemble (baseline + LGBM + dow safety)** | **29.37%** | **13.6%** | 304 unit | +62 |

**솔직한 결론**:
- **수치 정확도(WAPE)만 보면 단순 평균이 더 정확**
- v4의 진짜 가치는 **WAPE 외 영역**

### 1-2. 왜 단순 평균이 WAPE 더 낮은가?

| 이유 | 설명 |
|---|---|
| 광교 매장 = dow 패턴 매우 강한 매장 | 주말 +30% 고정, 평일 ±5% — 단순 평균만으로도 강한 예측 |
| 광교 5년 데이터 ceiling | 외부 features (날씨/특수일) 의 추가 신호 작음 |
| 평일 sample n 작음 (3-8개) | LGBM variance 큼, 단순 평균은 robust |
| 우리 Ensemble은 quantile production | 매진 회피 위한 over-prediction → WAPE 자연 ↑ |

## §2. v4 Framework의 진짜 부가가치 (Naive 대비)

### 2-1. 매진 safety buffer ★

| 차원 | Naive | v4 Ensemble |
|---|---|---|
| 매진율 (production < actual) | **45.5%** | **13.6%** |
| 의미 | 2일에 1일 매진 — 가정 1-2-b 위반 | 7일에 1일 매진 |

Naive는 point estimate (평균) → 50% 확률로 actual보다 작음 → 매진 회피 불가.
v4는 quantile 0.90 + dow safety → 통계적 매진 회피.

**비지니스 영향**: 인기품 조기 매진 → 고객 만족도 ↓ → 장기 매출 위험 회피.

### 2-2. 품목별 자동 분배 (Stage 2)

| 출력 | Naive | v4 |
|---|---|---|
| Stage 1 (카테고리 합) | ✓ | ✓ |
| **Stage 2 (품목별 qty)** | **X** | **✓** |
| Strong Up/Down/HOLD 자동 식별 | X | ✓ (분위수 가중 boost) |
| 신제품 1.2× boost | X | ✓ |
| 트렌드 ±20% boost | X | ✓ |

발주 시스템에 직접 연동 가능한 건 **품목별 qty**. Naive는 카테고리 합만 제공.

**광교 예시 (130개 품목 자동 분류)**:
- STRONG_UP: 올리브치아바타 (10.9시 매진, 추세 +28%)
- STRONG_DOWN: 식빵의정성 (마감 13%, 추세 -30%)

### 2-3. 신제품 자동 의사결정 (Stage 3)

| 차원 | Naive | v4 |
|---|---|---|
| 신제품 진입 시 발주량 | 별도 처리 X | **자동 1.2× boost** |
| 4주 후 정규/단종 결정 | 매니저 수동 | **자동 (적중률 76.3%)** |

광교 5년 81개 신제품 후행 검증:
- promote 의사결정 60개 중 45개 survived (성공률 75%)
- hold 의사결정 16개 중 9개 survived (안전 buffer)

### 2-4. 외부 features 통합 인프라

| Feature 그룹 | Naive | v4 |
|---|---|---|
| dow/월 자기상관 | ✓ | ✓ |
| 날씨 (avgTa, sumRn 등) | X | ✓ (importance 30%) |
| 캘린더 (휴일, before_holiday) | X | ✓ |

| 경쟁점 일자별 active | X | ✓ |

**광교 단독에선 외부 features 효과 작음 (+1pp 정도)**. 단 다매장 도착 시 진짜 가치.

### 2-5. 도메인 가정 + 분류 정리

## §3. v4 모델의 한계 (광교 단독 데이터)

### 3-1. WAPE 30% — 광교 5년 데이터 ceiling

| 원인 | 영향 |
|---|---|
| 평일 sample n 작음 (D+5 ~ D+7 fold당 3-8개) | LGBM variance 큼 |
| 외부 features 효과 작음 (광교 단독) | 학습 부족 |
| 외생 이벤트 정보 부재 | spike 예측 불가 |

### 3-2. 토요일 매진율 50% (D+9)

광교 토요일 매출 변동 큼 (±60 unit). 5년 데이터에서:
- 10개 토요일 중 4개 매진 (production < actual)
- 큰 fit-fail 사례: -46 unit (2025-05-24), -64 unit (2025-09-04)
- 우리 features로 설명 불가 (캘린더/특수일/날씨 모두 normal)
- → **외부 이벤트 spike** (마케팅, 신상 출시, 지역 행사) 추정

### 3-3. LGBM의 정확도 부가가치 작음

| 비교 | WAPE |
|---|---|
| Baseline (target_dow 평균) | 18.18% |
| Baseline + LGBM expected (residual) | 18.54% (+0.36pp) |
| 단순 평균 대비 LGBM 부가가치 | **거의 X** |

광교 단독 = 자기상관 lag만으로 강한 신호. LGBM이 추가로 잡을 신호 적음.

### 3-4. fade_out 룰 광교 미발현

신제품 81개 중 fade_out 의사결정 **0회**. 광교 데이터에서 `avg_sold < cat_median × 0.2 + 마감 ≥ 40%` 조건 동시 충족이 없음. 다매장 도착 시 룰 완화 필요.

### 3-5. 본질적 ML 한계

> 모델은 features 변동 → 출력 변동 매핑만 학습. **외부 이벤트(마케팅, 신상, 지역 행사)는 features에 들어가야만 학습 가능**.

광교 토요일 +30% spike의 원인이 마케팅이면 features에 그 정보 없으면 ML은 본질적으로 예측 불가능.

## §4. 추가 데이터 필요성 + 기대 개선 폭

### 4-1. 데이터 우선순위

| 순위 | 데이터 | 영향 | WAPE 개선 | 매진율 개선 |
|---|---|---|---|---|
| **1** | **본사 마케팅 / 신상 출시 캘린더** | ★★★ | **-5~8pp** | **-15~25pp** |
| **2** | **다매장 데이터 (2~5개 매장 5년치)** | ★★★ | **-3~5pp** | **-10~20pp** |
| **3** | **폐기 실측 (또는 입고량)** | ★★ | -1~2pp | 작음 |
| **4** | **광교 자체 promo / 행사 일지** | ★★ | -2~3pp | -10~15pp |
| **5** | **사전 예약 정보 (cake, 홀케이크)** | ★ | -0.5pp | 작음 |
| **6** | **채널 정보 (배달 vs 매장 픽업)** | ★ | -1~2pp | -2~3pp |
| **7** | **영업시간 / 휴무일** | ★ | -1pp | 작음 |
| **8** | **품목 무게 / 사이즈 / 칼로리** | ★ | -0.5pp | 작음 |
| **9** | 아브뉴프랑 입주사 행사 일정 | ☆ | -0.5pp | -2pp |
| **10** | 인근 시설 행사 (광교호수공원 등) | ☆ | -0.5pp | -2pp |

### 4-2. 데이터별 상세

#### 1순위. 본사 마케팅 / 신상 출시 캘린더 ★★★

**왜 필요한가**:
- 토요일 매출 spike (+30%) 원인의 가장 큰 후보
- 신상 출시일 ±7일 매출 boost +20~30% 캡처 필요
- 본사 캠페인 (1+1 행사, 쿠폰 발급) 효과 캡처

**필요 schema**:
```
campaign_date    : 행사 일자 (또는 시작일~종료일)
campaign_type    : new_product / promotion / coupon / collaboration
scope            : national / store_specific (광교만)
expected_lift    : 매출 부스트 예상 (선택, 5-50%)
target_category  : bread / pastry / all
```

**기대 효과**:
- 토요일 fit-fail outlier 캡처
- WAPE -5~8pp (현재 29% → ~22%)
- 매진율 -15~25pp (현재 14% → ~3%)

#### 2순위. 다매장 데이터 ★★★

**왜 필요한가**:
- 광교 5년 = 단일 매장 dow 패턴 의존도 너무 큼
- 매장 간 transfer learning → LGBM 진짜 가치 발휘
- 외부 features (경쟁점, 유동인구, 인구통계) 효과 본격 측정
- fade_out 룰 광교 미발현 → 다른 매장에서 검증

**필요**:
- 광교 외 2~5개 매장
- 같은 5년 (또는 가능한 만큼)
- 동일 schema (sold, stockout, closing 등)

**기대 효과**:
- WAPE -3~5pp (모든 매장 평균)
- 매진율 -10~20pp (특히 토요일 dow effect 학습)
- framework 일반화 가능

#### 3순위. 폐기 실측 ★★

**왜 필요한가**:
- 현재 α=0.6 휴리스틱 — 데이터 기반 정확화 필요
- 진짜 ROI 측정 (보수 추정 2.3M원 → 실측 5-15M원 추정)
- 폐기 cost 직접 측정 (현재 마감 할인 절감만 측정)

**필요 schema**:
- Option A: `(date, item_id, waste_qty, waste_cost)` 직접
- Option B: **입고량** `(date, item_id, ingestion_qty)` — 광교 95% 당일폐기라 `폐기 = 입고 − 판매` 자동 계산 가능
- Option C: 시작/종료 재고 (가장 정확하지만 받기 어려움)

**기대 효과**:
- α 캘리브레이션 (현재 0.6 → 데이터 기반 0.5~0.7)
- WAPE -1~2pp
- 비지니스 KPI 정확도 (Waste rate 일 20개 → 8개 정확 측정)

#### 4순위. 광교 자체 promo 일지 ★★

**왜 필요한가**:
- 본사 캠페인 외 광교 매장 자체 운영 결정 (예: 시간대 할인, 매장 발행 쿠폰)
- 매장 매니저 운영 노트

**필요 schema**:
```
date            : 일자
promo_type      : time_discount / coupon / event
description     : 자유 텍스트
expected_lift   : (선택)
```

**기대 효과**:
- 광교 특화 매출 변동 캡처
- WAPE -2~3pp
- 매진율 -10~15pp (특히 토요일)

#### 5순위. 사전 예약 정보 (cake) ★

**왜 필요한가**:
- 광교 케이크 매출의 대부분이 사전 예약일 가능성 (현재 cake 카테고리 Stage 1 제외)
- 예약 vs 일반 판매 분리해야 cake 합산 가능

**필요 schema**:
```
reservation_date : 예약 받은 날짜
pickup_date      : 인도 일자
item_id          : 예약 품목
qty              : 예약 수량
amount           : 예약 금액
```

**기대 효과**:
- cake 카테고리 Stage 1 합산 가능 (현재 제외)
- WAPE -0.5pp (cake 비중 2%라 직접 영향 작음)
- cake 시즌 특수 모델 가능

#### 6순위. 채널 정보 ★

**왜 필요한가**:
- 배달 채널 (배민, 쿠팡잇츠 등) vs 매장 픽업 분리
- 채널별 hour profile 다름 (배달은 점심 집중, 매장은 분산)
- 마감 할인 vs 정가 분리에도 영향

**기대 효과**:
- WAPE -1~2pp
- 채널별 demand 분리 가능

#### 7~10순위. 작은 효과지만 데이터 정리 가치

| 데이터 | 효과 |
|---|---|
| 영업시간 / 휴무일 | open_hour/close_hour dynamic → potential_demand 정확도 |
| 무게 / 사이즈 / 칼로리 | 진짜 정규화 unit (kcal 또는 1회분량 매핑) |
| 아브뉴프랑 입주사 행사 | 지역 매출 boost |
| 인근 시설 행사 | 광교호수공원 / 컨벤션센터 행사 |

- **단순 dow 평균과 큰 정확도 차이 X** (WAPE 측면)

- ML 알고리즘 튜닝만으론 WAPE 5-10pp 개선이 ceiling

### 6-2. 진짜 가치는 framework 자체

| 가치 | 의미 |
|---|---|
| **매진 회피 자동화** | 45% → 14% |
