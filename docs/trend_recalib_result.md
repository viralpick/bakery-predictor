# 추세 recalibration 진단 결과 (트랙1) — 광교 3년 OOS

이상치 분석(2026-07-16)에서 도출한 모델 개선 4트랙 중 **1번(추세 미추종 recalibration)** 검증.
가설: 광교 빵류 수요가 2022 정점 후 하락(일평균 319→236, −26%) → 730d 롤링 학습이 지연추종
→ 하락기(특히 2023) 상시 과대예측 → 폐기 편향.

## 결론: **트랙1 drop** (이 target에서 순가치 없음)

target 재정의(α=0.8·bulk 제외·EventLevelPrior)가 이상치 분석 당시의 지연추종 편향을
이미 대부분 흡수했다. 남은 편향은 작고, 창 단축·온라인 보정 어느 쪽으로도 폐기-매진
파레토 개선을 못 만든다. 온라인 보정은 오히려 단순 발주 하향보다 열등.

## 근거

### (a) 창 단축 — window×연도 WPE (편향, +=과대예측, %)
| window | 2023 | 2024 | 2025 | 판정 |
|--------|------|------|------|------|
| 180d | 1.63 | **0.21** | −0.33 | 2024만 개선 |
| 365d | 2.26 | 2.07 | 0.14 | |
| 730d (현행) | 1.67 | 1.62 | 0.64 | |
| 1095d | 1.54 | 1.35 | 0.90 | |
| 1825d | 1.54 | 1.21 | 1.11 | |

- 편향 자체가 작음(연간 max 1.67%). 긴 창일수록 최근연도 편향이 양(+)으로 지속(지연추종 지문),
  짧은 창은 2024 편향↓ — 단 2023 아님. **WAPE는 창별 ±0.3pp로 중립.**
- 헤드라인 window 변경 시 EventPrior·N·K 버퍼 전부 재튜닝 비용 → 2024-only 소폭 이득 대비 과함. **미채택.**

### (c) 온라인 레벨 보정 — 730d 유지 + 직전 M주 편향 곱셈 보정
연간 WPE (baseline 1.67/1.62/0.64 → online_M8 0.16/0.31/0.17)는 예뻐 보이나:

**월별 WPE spread (baseline vs online_M8)**
| | mean\|WPE\| | std | worst\|·\| |
|---|---|---|---|
| baseline | 2.26% | 2.49% | 6.72% |
| online_M8 | 1.91% | **2.49%** | 6.56% |

- 시기집중 편향은 reproduce(최악 월 6.72% vs 연간 1.67% — 연간은 상쇄로 작아 보였을 뿐).
- 하지만 online이 그 swing을 **못 잡음**: std 그대로, turning point 과보정으로 최악 과소달 −3.06→−6.56% 악화(reactive lag).

**폐기·매진 (production=q0.85)**
| variant | 폐기율 | 매진율 |
|---|---|---|
| baseline | 9.18% | 21.52% |
| online_M8 | 8.35% | 26.88% |

- 폐기 −0.83pp이나 매진 +5.4pp — production 레벨 전반 하향(발주 공격성↓)과 구분 안 됨.

**iso-waste 최종 판정 (폐기율 8.35%로 통일)**
- baseline×0.99 (상수 발주하향): 매진율 **25.18%**
- online_M8 (시간가변 보정): 매진율 **26.88%**
- → **상수 하향이 우위.** 시간가변 추세 보정은 그냥 발주 1% 하향보다도 못함 = 순가치 음(-).

## 함의 (모델 개선 4트랙)
- **잔차의 모양이 방향을 가리킨다 (가장 깨끗한 신호):** online 보정 후 월별 WPE **std가 2.49%로 불변**
  (mean|WPE|만 2.26→1.91 하락). 추세추종 보정이 제거한 건 느린 레벨 오프셋(작음)뿐 — 최악 월(std 2.49,
  worst 6.72%)을 만드는 성분은 추세추종을 **통과**한다 = **추세형이 아니라 이벤트/계절형**. 즉 남은 편향은
  트랙1엔 잘못된 모양이고, EventLevelPrior가 겨냥하는 **트랙2엔 맞는 모양**이다. 트랙1→2 전환은 "안 됐으니 다음"이
  아니라 "잔차 구조가 트랙2를 가리킴".
- **(2026-07-18 후속 확증)**: 트랙2에서 헤드라인 공휴일 feature가 2021-23 blind였음이 드러남
  (`docs/holiday_premium_decomposition.md`). 이는 트랙1의 "잔차=추세형 아닌 이벤트/계절형" throughline을
  **소급 확증** — 그 이벤트/계절 잔차의 일부가 이 공휴일 blindness였다. 트랙1 drop 유지(오히려 강화).
- **트랙1 = 종료.** 편향이 이미 작아 recalibration 레버 무효. 다음 세션은 **트랙2(명절 prior 확장)** — 편향이
  실제로 큰 곳(명절 12일)이라 임팩트 명확하고 기존 EventLevelPrior 메커니즘 재사용(낮은 난이도).
- (참고) 수요 −26% 하락은 여전히 실재하나, "레벨 추종"으로 짜낼 이득은 이 target에선 소진. 고객사 구조 문의(경쟁·상권)가 본질.
