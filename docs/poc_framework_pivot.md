# 베이커리 PoC — 핵심 발견과 모델링 framework 재정의

> 광교 매장(아티제 아브뉴프랑광교점) 보나비 5년치 데이터 + 영수증 22만건 cross-sell + Substitution RD/MNL/DiD 분석 종합.
> Notion 공유용. 기존 v1/v2 리포트의 일부 가정이 데이터 검증 결과 틀린 것으로 확인 — 모델링 방향 전환 권고.

---

## TL;DR (5줄 요약)

1. **광교 베이커리는 손님이 "카테고리 = 한 묶음"으로 보고 사가는 매장**. 특정 품목 집착 거의 없음 (cake 등 시즌 제품 제외).
2. **품절 → 매출 손실** 가정은 데이터에서 확인되지 않음. 카테고리 내 거의 100% 흡수 → 매장 매출 보존됨. 기존 substitution 모델(RD outflow 55%)은 confound가 부풀린 결과.
3. **진짜 risk는 ①카테고리 전체 supply 부족 ②선택지 너무 빈약 ③시즌 특수 제품 품절** — 이 3개로 PoC 모델링 framework를 재정의해야 함.
4. **이미 갖고 있는 할인/프로모션 데이터(34% row + 마감 시간대 36% 집중)가 현재 모델에 미반영** (§6). 활용하면 hour profile · substitution · 폐기 회수 분석 정확도 크게 향상.
5. **부족한 데이터 11개 우선순위** (§7). 1순위는 폐기 수량(또는 입고량으로 회계 항등식 유추), 다음으로 채널 구분·예약 정보·영업시간.

## 1. 기존 가정 vs 데이터 검증 결과

### 기존 가정 (PoC v1/v2)

```
A빵 품절 → 손님 일부는 B빵 구매 (substitute)
            손님 일부는 매장 떠남 (영구 매출 손실)
            cross-sell 손실까지 가중 (multiplier 1.7×)
                        ↓
            잃은 매출 24개월 환산 2.50억원
            v2_q85 production 도입 시 net_profit +21M
```

### 데이터 검증 (광교 5년)

| 검증 항목 | 결과 | 의미 |
|---|---|---|
| 매장 매출 (i_early 일 vs 정상일) | 279.9 vs 276.9 (1% 차이) | **떠나는 손님 거의 없음** |
| DiD β_iearly (FE + traffic + j_lag 통제) | 평균 ≈ 0, p<0.05 양수 43개 / 음수 52개 | 개별 i→j substitute 시그널 거의 없음 |
| MNL Spearman vs RD | 0.248 (약한 양상관) | 두 모델이 다른 신호 잡음 — 진실 식별 어려움 |
| Nested logit λ_g | 0.99~0.995 | 카테고리 nest 경계 약함 |
| RD outflow 평균 | 55% (cap 0.7) | confound 큼 — DiD가 통제 후 사라짐 |
| **할인 row 비율** | **34.1%** (광교 단품·정상매출 45만 중 15만) | 모델에 미반영 — 분석 결과 noise 가능 |
| **마감 시간대(20~21시) 할인 집중** | **전체 할인의 36%** | 마감 할인 = 폐기 회수. "품절"이 사실 일부는 회수된 판매일 수도 |

### 결론

데이터는 다음과 같이 말함:
- **i 품절은 매출 손실로 이어지지 않음** (적어도 광교의 일반 빵 카테고리에서)
- 손님은 카테고리 안에서 자유롭게 흡수됨 — 개별 i↔j 페어 단위 substitute는 측정할 만큼 강하지 않음
- 영구 매출 손실 추정 (RD 기반 2.50억원)은 **과대 추정**일 가능성

## 2. 사용자 통찰 — 베이커리 demand의 본질

> "베이커리 온 사람은 '빵' 그 자체가 목적. 바게트 없으면 팥빵 사가는 게 아니라, 그냥 남아있는 빵 중 본인이 필요한 양만큼 사감.
> 정작 risk는 그날 전체 품목 품절 또는 선택지 너무 빈약한 케이스."

**이를 모델 가정으로 표현**:

```
손님 의사결정 단위:  "빵 카테고리" (특정 품목 아님)
                        └─ 카테고리 내 잔여 품목 중 임의 선택 (취향 + 운반성 + 가격)
                        └─ 카테고리 supply > 0인 한 매출 발생

진짜 risk 발생:
                        ① 카테고리 전체 supply 부족 (예: 빵 카테고리 모두 품절)
                        ② 선택지 빈약 (남은 품목 종류가 2~3개 이하)
                        ③ 시즌 특수 제품 (cake, 발렌타인 sweets, 명절 선물 등)
                           — 손님 목적이 특정 품목이므로 진짜 substitute 어려움
```

이 framework는 기존 인사이트들과 어떻게 맞물리는가:

| 기존 인사이트 | 새 framework에서의 위치 |
|---|---|
| 1. Self-fulfilling stockout | 카테고리 단위에선 매출 보존 → 영구 손실 작음. *진짜 가치는 "폐기 비용 절감"이지 "매출 회복"이 아님* |
| 3. Cross-sell 손실 (1.7×) | 매장 떠나는 손님이 적으므로 1.7× multiplier 적용 대상 자체가 작음. 광교 영수증 평균 1.86 품목은 일관적 사실이지만 *손실 가중* 해석은 약화 |
| 6. Substitution outflow 55% | RD confound가 부풀린 추정. DiD 통제 후 effect 거의 0. 진실은 "카테고리 내 거의 100% 흡수" |
| 7. Inter-category sub > within | 카테고리 경계가 약함 (인기 품목 끼리 흡수). 새 framework와 일관 |
| 9. Nested logit λ≈0.99 | "베이커리에서 IIA 거의 안 어김" = "카테고리 nest 경계 약함" = 새 framework와 같은 결론의 다른 측면 |

## 3. 새 모델링 framework

### 3.1 의사결정 단위 재정의

```
1차 의사결정 (사업 KPI):  카테고리 단위 supply 결정
                            └─ "오늘 빵 카테고리 N개 만들자"
                            └─ supply ≥ demand만 유지하면 매출 보존

2차 의사결정 (운영 효율):  카테고리 내 품목 mix
                            └─ "빵 N개 중 어떤 비율로 나눌까"
                            └─ 폐기 비용 절감용 (인기 품목 많게, 비인기 적게)

3차 의사결정 (시즌 특수): 특수 제품 (cake / 시즌 sweets) 품목 단위 정밀 예측
                            └─ 진짜 substitute 어려움 → 정확도 중요
                            └─ 발렌타인·크리스마스·생일 케이크 등
```

### 3.2 매출 손실의 진짜 source

| 손실 종류 | 광교 실제 크기 (재추정) |
|---|---|
| ~~품절 → 매장 떠남~~ | **거의 0** (매장 매출 보존됨) |
| **품목 mix 비효율 (인기 underproduce, 비인기 overproduce)** | **폐기 비용 ↑** — v2/v3 production model의 진짜 가치 |
| **카테고리 전체 supply 부족** | 진짜 영구 손실 (드물지만 발생 시 큼) |
| **시즌 특수 제품 품절** | 진짜 영구 손실 (대체 어려움) — cake 등 |
| **마감 할인 회수율** | 광교 할인의 36%가 20~21시 집중 = 폐기 직전 회수. 진짜 폐기 손실은 cost × (1 − 회수율) |

### 3.3 모델링 방향 전환 후보

**옵션 A — Category-level forecast (top-down)**
- 카테고리당 일별 총 매출 예측 모델
- 카테고리 내 품목 share는 단순 baseline (요일별 인기 비중) 또는 매장 운영자 경험
- v2_q85 quantile은 **카테고리 단위**로 적용 → "오늘 빵 N±X개 만들자"

**옵션 B — Hierarchical (현재 모델 + 카테고리 layer)**
- 현재 v0~v3 품목 단위 예측 유지
- 카테고리 단위 aggregation layer 추가 → KPI는 카테고리 단위
- 두 차원 모두 보존

**옵션 C — 현재 모델 유지 + 해석·KPI만 재정의**
- 코드 변경 없이 사업 KPI 재정의
- 잃은 매출 추정 framework를 카테고리 단위로 갱신
- v2_q85의 진짜 가치를 "폐기 비용 절감"으로 재포장

권고: **옵션 C → A** 순차 진행. C로 즉시 결론 정렬, A는 다매장 도착 시 본격 구현.

## 4. PM 평가 지표와 현재 PoC의 갭

PM 담당자 기획서의 평가 지표 framework:

### (1) 점 예측

| 지표 | 설명 | 판단 기준 | 현재 PoC 상태 |
|---|---|---|---|
| **wMAPE** | 매출 가중 평균 오차율 | 0.12 = 12% 오차 | ✅ 측정 중 (`wape_all` 컬럼). 광교 v0: 0.215, v2_q85: 0.352 — 목표 0.12 대비 약함. 단 v2_q85는 target이 potential_demand라 wMAPE 단독 평가 부적합. **카테고리 단위 wMAPE 신규 측정 필요** |
| **MASE** | seasonal_naive 대비 개선율 | < 1 |  ❌ **미구현** — backtest summary에 추가 필요 |

### (2) 구간 예측

| 지표 | 설명 | 현재 PoC 상태 |
|---|---|---|
| **Coverage@80% / @95%** | "80% 신뢰구간"이 진짜 80% 들어와야 신뢰 | ❌ **미구현** — production quantile 모델 신뢰성 검증 안 됨. α=0.85 → coverage 측정 시 80%, 85%, 90% 어디 들어가는지 확인 필요 |
| **Pinball Loss** | quantile regression 표준 loss | ⚠️ LightGBM 내부 학습 loss로 사용 중이나 **보고 안 함** |

### (3) 최종 검증 (운영 KPI)

| 지표 | 목표 예시 | 현재 PoC 상태 |
|---|---|---|
| **Waste rate (폐기율)** | 일 평균 20개 → 8개 (60% ↓) | ⚠️ `waste_cost` KPI에 포함되어 있으나 **단위가 비용(원)이지 개수가 아님**. 폐기 개수 명시적 측정 필요 |
| **Stockout rate / hours** | 1시간 → 15분 (75% ↓) | ⚠️ `is_stockout`은 binary, `stockout_time`은 시각 — **품절 지속 시간 미측정** |
| **손실 매출 절감** | 15만원/일 → 5만원/일 | ⚠️ `lost_margin` KPI 있으나 새 framework에서는 **카테고리 단위로 재정의 필요** (개별 품목 단위 추정은 과대 가능성) |
| **발주 적중률** | 권장 범위 내 실제 판매 비율 | ❌ **미구현** — production model 권장량 ± 신뢰구간 안에 실제 sold가 들어오는 비율 측정 안 됨 |

### 갭 요약

| 미구현 항목 | 우선순위 | 작업량 |
|---|---|---|
| MASE 측정 | High | 작음 (10줄) |
| Coverage@80, @95 | **High** (production model 신뢰성) | 중간 (50줄) |
| Pinball Loss 보고 | Medium | 작음 (5줄) |
| Waste rate (개수 단위) | Medium | 작음 |
| Stockout hours (지속 시간) | Medium | 중간 |
| 발주 적중률 | **High** (사업 핵심 KPI) | 중간 |
| 카테고리 단위 wMAPE | **High** (framework 전환 결과) | 중간 |

→ 위 7개 모두 framework 재정의와 함께 다음 단계에서 구현 필요.

PoC 데이터 재점검에서 발견: 보나비 xlsx에 할인 데이터가 풍부히 있으나 현재 v0~v3 모델 어디에도 반영되지 않음.

| 분석 항목 | 현재 영향 | 할인 데이터 반영 시 |

| **Hour profile** | 마감 할인이 진짜 demand 패턴으로 오해됨 | 정상 매출 hour profile 정제 |
| **potential_demand 보정** | 마감 할인 매출 = 진짜 demand로 학습 | 보정값 더 정확화 |
| **DiD / RD substitution** | 마감 할인 효과를 substitution으로 잘못 잡을 가능성 | 진짜 substitution 시그널 정제 |
| **자기실현적 사이클** | "품절"이 일부는 사실 할인 회수된 판매 | 진짜 영구 매출 손실 더 작아짐 |
| **사업 KPI net_profit** | cost_rate=0.30 일률 가정 | 회수율 기반 정밀 cost 추정 가능 |
| **카테고리=한 묶음 framework** | 일반 매출만 보면 OK | 마감 할인 + 일반 매출 분리로 framework 강화 |

### 6.5 즉시 가능한 작업

코드·인프라가 이미 준비되어 있어 다음 작업은 모두 1~2일 내 가능:

1. **할인 row 분리**: `is_discounted` 컬럼 추가 → 정상/할인 매출 분리 학습
2. **마감 할인 feature**: 시간대 × 할인 여부 cross feature → 자기실현적 사이클 정밀화
3. **할인코드 의미 매핑**: 30개 광교 사용 코드 → 카테고리(마감세일/예약/제휴/회수/이벤트) 분류
4. **회수율 추정**: `(마감 할인 매출 unit) / (당일 폐기 직전 unit) ≈ 회수율`
5. **케이크 예약 row 분리**: 할인코드 "(케이크예약)" 패턴으로 예약 매출 식별 → 시즌 특수 모델 분리

### 7.1 폐기량 유추 — 회계 항등식 (1순위 옵션)

폐기수량 직접 받기 어려운 경우:

```
시작재고 + 입고 = 판매 + 폐기 + 종료재고
→ 폐기 = 입고 + 시작재고 − 종료재고 − 판매(기존 보유)
```

광교는 당일폐기 Y가 397/417 (95%) → 다음날 이월 0 → **`폐기 = 입고 − 판매`**.
**일별 입고량 하나만 받으면 95% 품목 폐기량 정확 계산 가능**. 발주·제조 시스템에 이미 있을 가능성 큼 (매장 운영자 매일 작성).

## 8. 솔직한 PoC 자가평가

### 이번 PoC에서 잘된 것

- 인프라 완성도 (외부 데이터 4축 ingest, 시간순 backtest, 사업 KPI)
- 다양한 모델 비교 (v0~v3, baseline, production quantile)
- substitution 정량화 시도 (RD, MNL, Nested, DiD) — 비교를 통해 confound 발견
- **자기 가설을 데이터로 검증한 결과 framework가 틀렸음을 인지** ← 이게 가장 큰 발견

### 솔직히 부족한 것

- 단일 매장 데이터의 본질적 power 한계 — substitution 식별 자체가 어려움
- PM 평가 지표 framework와의 정렬 부족 (MASE, Coverage, 적중률 등 미구현)
- 영수증 카테고리 패턴 직접 검증 안 함 (옵션 5)
- 잃은 매출 추정이 처음부터 과대 가능성 — 사업 임팩트 추정의 신뢰도 보정 필요
- **할인/프로모션 데이터를 갖고 있으면서 모델에 미반영** (§6) — 34% row가 할인 매출인데 정상 매출과 섞어 학습
- **폐기 수량 실측 부재** — PM 평가 지표 Waste rate를 직접 측정 못함 (§7.1로 우회 가능)

### 핵심 메시지

> **PoC가 모델 정확도를 약간 올리는 것보다, "베이커리 demand의 본질을 잘못 짚었던 가정"을 데이터로 식별한 것이 더 큰 가치.**
>
> 다음 단계는 정확도 올리기 경쟁이 아니라 **카테고리 단위 demand framework**로 모델·KPI·운영 가이드를 재정렬하고, PM 평가 지표 체계로 사업 의사결정을 명확하게 만드는 것.
>
> **이미 갖고 있는 데이터(할인/프로모션)부터 활용**하고, **부족한 데이터(폐기/입고/예약/채널)는 §7 우선순위 순으로 보충**하면 PoC를 완성도 있게 마무리 가능.
