>
> 킥오프 액션아이템(A1~A10) 매핑은 각 항목에 표기.

> **🔒 내부 전략 메모 (메일 발송 X)**
> - **현행 baseline의 약점 = 품목별(item-level) 컨트롤**. 수요 흡수(카테고리 내 대체)를 고려하지 않고 품목마다 sold-out 지수로 증감산하면 노이즈가 크고 실제(흡수 발생)와 맞지 않아 오차가 클 것으로 판단. → 이것이 **우리 모델 우위 가설의 핵심 근거**이며, 항목 1(발주 로직)을 확보하려는 진짜 이유는 이 item-level 로직을 우리 **카테고리 합 → 품목 비율 배분(v4)** 방식과 diff 하기 위함. (근거: v4 프레임워크, W0 흡수검증)
> - **우리 레버 = 전체 생산량 예측 후 비율 조정(Stage2 ALLOCATE)의 정교화.** 데이터 요청이 아닌 모델링 TODO. 여기서 유의미한 개선 여지.
> - **당일 생산·당일 폐기 확정 → weekly 생산량 맞추기 없음. daily 예측이 결정적.** 따라서 모든 요청 데이터(특히 발주량)는 **일(日) 단위**로 받는다.

## 통합 요청 목록 (9항목)

우선순위 순. 시스템에서 추출 가능한 범위·형식에 맞춰 조정 가능.

### ■ 최우선 — 전향 실측 비교의 baseline 직결

#### 1. 현행 발주 시스템 로직  〈A1〉

- **왜**: 4주 전향 실측에서 "기존 발주 방식 vs 우리 모델"을 비교하는데, **기존 방식이 정확히 무엇인지** 알아야 baseline이 선다. 현재 이 로직이 우리 데이터에 전혀 없다.
- **요청 형식**: 로직 설명 문서 또는 산식. 특히 —
  - sold-out 지수 정의·계산식 (과거 **3주** 참조 맞는지)
  - **요일별** 보정 방식, 그 외 반영 인자
  - 시스템이 산출하는 **발주 제시량**의 계산 흐름 (예측량 입력 → 발주량 자동 산정 경로 포함)

#### 2. 일별·품목별 발주량  〈A5〉

- **왜**: 현행 시스템의 실제 발주량을 알아야 (a) baseline 성능을 직접 측정하고 (b) 예약(bulk) 검출 아이디어 ②(**판매+폐기 > 발주량 = 추가발주 = 예약**)를 검증할 수 있다. **판매량·폐기량과는 별도 컬럼**으로 필요.
- **⚠️ 함께 확인 요청**: 발주는 목요일에 이뤄진다고 이해했는데, **발주일 이전에 들어온 사전 예약이 시스템 발주량에 이미 반영된 채로 발주되는지** 확인 필요. 반영 여부에 따라 예약분 분리 로직이 달라진다.
- **요청 형식**:

  | 컬럼 | 타입 | 예시 |
  |---|---|---|
  | 점포코드 | string | 1000000047 |
  | 발주일자(또는 입고예정일) | date | 2024-01-15 |
  | 품목코드 | string | 151100000247 |
  | 발주수량 | int | 24 |

  가능하면 발주 시점(요일)·확정/조정 여부 컬럼도.

### ■ 발주 정확도 — 반올림·제약 반영

#### 3. 상품별 생산단위(배수) 룰  〈A3〉

- **왜**: 냉동생지 1개 = N개 분량이라 **3/6/9 배수로만 생산** 가능하다고 확인했다. 우리 발주량 산출 뒤 **품목별 배수 제약으로 반올림**해야 실행 가능한 수치가 된다.
- **요청 형식**: `(품목코드, 최소 생산단위 / 배수)`. 예: 품목 X = 3개 배수, 품목 Y = 6개 배수, 완제품 입고품 = 1(제약 없음).

#### 4. 품목별 제품 유형  〈A4〉

- **왜**: 유형마다 발주·생산 방식이 달라 모델의 발주 반올림·lead time 처리가 달라진다.
- **요청 형식**: `(품목코드, 제품유형)`. 유형 예시 —
  - ① 완제품 입고 (정수량 발주)
  - ② 직접 생산 (매장 제조)
  - ③ 냉동생지 구움 (배수 제약)
  - ④ 생지 분할 (배수 제약)

### ■ 향후 운영 반영

#### 5. 신제품 목록·출시일 (향후)  〈A6〉

- **왜**: 신제품은 반응을 보려 **덜 팔려도(폐기 감수) 넉넉히 배치**한다는 정책을 확인했다. 모델이 신제품 구간에 폐기 페널티를 낮추려면 **출시 예정일**을 알아야 한다. 앞으로 신제품 출시가 많다고 하여 중요.
- **요청 형식**: `(품목코드/품목명, 출시예정일, 카테고리)`. **과거 신제품 이력은 데이터에서 유추 가능하므로 "향후 출시" 목록 위주로** 부탁.

#### 6. 프로모션 일정 (향후)  〈A8〉

- **왜**: 프로모션 기간의 수요 변동을 feature로 반영. 과거 프로모션은 할인코드에서 유추 가능하나, **향후 일정**은 사전에 받아야 예측에 반영된다.
- **요청 형식**: `(프로모션명, 기간, 대상 품목/카테고리, 할인 방식, 적용 매장)`.

### ■ 비용 정량화

#### 7. 원가율 / 마진  〈기존 갭 유지, A 매핑 없음〉

- **왜**: 폐기·매진 비용을 **금액으로 정량화**하고 발주 안전마진(분위수)을 비용 기반으로 최적화하려면 원가율이 필요하다. 현재는 발주 패턴 역산으로 ~0.20(빵)/0.28(페이스트리) 수준으로 **추정**만 하고 있다.
- **요청 형식**: 품목별 원가율/마진이 최선이나, **세부 공개가 어렵다면 빵 카테고리 전체의 대략적인 원가율 수치라도** 도움이 된다.

### ■ 장기 · 선택

#### 8. 지역 이벤트 캘린더 / 품목별 진열 시각  〈A7·A9〉

- **왜(이벤트)**: 수원 월드컵경기장 행사(특히 **싸이 흠뻑쇼**) 때 광교 애비뉴프랑 인파가 몰려 매출이 급증한다고 확인했다. 광교 특화 지역 이벤트가 명절보다 클 수 있어, **점장이 이미 보유·인지하는 이벤트 목록**이 있으면 반영 가치가 크다.
- **왜(진열 시각)**: 제품마다 진열 시작 시각이 달라(개장부터 다 있는 게 아님) 시간대 판매·품절 해석을 보정하려면 `(품목, 진열 시작 시각)`이 필요.
- **원칙**: 이벤트는 **아주 명확한 것만** — 세부까지 다 넣으면 운영 부담이 크다. 장기적으로 자동 수집(웹 → 승인 → feature)으로 전환 목표.

### ■ 분석 심화 — 고객 트래픽 추정 (강력)

#### 9. 기타 품목(음료 등) 판매 데이터  〈신규, cf. 카페 traffic 확장〉

- **왜**: 당일생산·당일폐기 베이커리 품목 외에, 매장에서 판매되는 **기타 품목(특히 음료)**의 판매 데이터. 아띠제 매장은 카페가 메인이라 **음료 판매 수 = 방문 고객 수 proxy**로 쓸 수 있다. 현재는 베이커리 영수증 건수로만 트래픽을 근사 통제 중.
- **분석 가치**: ① 수요를 고객 수로 정규화해 **"수요 흡수 vs 순수 트래픽 변동"** 분리 ② 매진→매출 분석에서 트래픽 통제 정밀화(early_share 내생성 완화) ③ 외부 이벤트(싸이 흠뻑쇼 등) 효과를 고객 수 기준으로 검증 → 킥오프 가설(매진 무영향)의 근거를 크게 강화.
- **요청 형식**: `(점포코드, 일자, 품목/카테고리, 판매수량)`. 베이커리 외 전 품목이 최선이나, 최소 **음료 카테고리**만이라도.

우선순위 순으로 정리했으며, 매장/본사 시스템에서 **실제 추출 가능한 범위·형식**에 맞춰 주셔도 됩니다. 일부만 가능하시다면 가능한 항목부터 주셔도 저희가 그에 맞춰 작업하겠습니다.

**［최우선 — 전향 실측 비교에 직결］**

1. **현행 발주 시스템 로직** — sold-out 지수 계산식, 참조 기간(3주), 요일 보정·기타 반영 인자, 그리고 예측량을 넣으면 발주량이 산정되는 흐름까지, 설명 자료로 공유 부탁드립니다.
2. **일별·품목별 발주량 데이터** — 판매량·폐기량과 **별도로**, `(점포코드, 발주일자, 품목코드, 발주수량)` 형태면 충분합니다.
   - 한 가지 확인 부탁: **발주일(목요일) 이전에 들어온 사전 예약분이 시스템 발주량에 이미 포함된 채로 발주되는지** 여부를 알려주시면, 예약분을 분리하는 로직을 정확히 세울 수 있습니다.

**［발주 정확도 — 실행 가능한 수량 산출］**

3. **상품별 생산단위(배수) 룰** — 어떤 품목이 3/6/9 등 배수로만 생산되는지 `(품목코드, 최소 생산단위)` 형태로 부탁드립니다.
4. **품목별 제품 유형** — 완제품 입고 / 직접 생산 / 냉동생지 구움 / 생지 분할 등, `(품목코드, 제품유형)` 매핑을 부탁드립니다.

**［향후 운영 반영］**

5. **향후 출시 예정 신제품 목록·출시일** — 과거 신제품 이력은 저희가 데이터로 유추 가능하니, **앞으로 출시 예정인** 품목과 출시일 위주로 주시면 됩니다.
6. **향후 프로모션 일정** — 기간·대상 품목·할인 방식·적용 매장. 마찬가지로 **향후 일정** 위주로 부탁드립니다.

**［비용 정량화］**

7. **원가율 / 마진** — 폐기·매진 비용을 금액으로 산출하는 데 필요합니다. 품목별이 어렵고 민감한 정보임을 이해하니, **빵 카테고리 전체의 대략적인 원가율 수치**만이라도 주시면 큰 도움이 됩니다.

**［장기 · 여유 되실 때］**

8. **지역 이벤트 캘린더 / 품목별 진열 시각** — 점장님이 이미 알고 계신 지역 행사(예: 수원 월드컵경기장 공연 등) 목록과, 제품별 진열 시작 시각이 있으면 시간대 분석 정확도가 올라갑니다. 이 항목은 여유 되실 때 주셔도 됩니다.

**［분석 심화 — 고객 트래픽 추정］**

9. **기타 품목(음료 등) 판매 데이터** — 당일생산·당일폐기 베이커리 품목 외에, 매장에서 판매되는 기타 품목(특히 음료)의 판매 데이터도 함께 주시면 분석에 큰 도움이 됩니다. 카페가 메인인 매장 특성상 **음료 판매 수를 방문 고객 수의 대리지표(proxy)**로 활용해 수요를 고객 수로 정규화하면, 트래픽 변동과 수요 흡수를 분리해 분석 정확도가 크게 올라갑니다. `(점포코드, 일자, 품목/카테고리, 판매수량)` 형식이면 충분하며, 최소 음료 카테고리만이라도 좋습니다.

번거로우시겠지만, 각 항목의 **추출 가능 여부와 대략적인 형식**만 먼저 회신 주셔도 그에 맞춰 데이터 규격을 확정해 다시 안내드리겠습니다.

감사합니다.
