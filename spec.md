# 베이커리 판매량 및 품절 예측 데이터 분석 기획안

## 0. 배경 및 목적

베이커리 매장의 과거 판매량 및 품절 데이터를 기반으로, 다음주 품목별 예상 판매량과 품절 위험을 예측한다.

이번 분석의 목표는 단순히 “판매량을 맞히는 모델”을 만드는 것이 아니라, 실제 매장 운영에서 활용 가능한 형태로 다음 의사결정을 지원하는 것이다.

 '''text
 - 다음주 품목별 예상 판매량 예측
 - 품목별 품절 위험 예측
 - 권장 생산량 또는 생산량 범위 제안
 - 날씨, 공휴일, 유동성, 상권, 연령대, 소비정보가 수요에 미치는 영향 분석
 - 외부 데이터가 실제 예측 성능 개선에 기여하는지 검증
 '''

특히 베이커리 데이터는 품절이 발생한 경우 실제 수요가 관측 판매량보다 높았을 가능성이 있기 때문에, 판매량 예측과 함께 품절 여부 및 품절 시각을 반드시 함께 고려해야 한다.

---

## 1. 분석 범위

### 1.1 4주 MVP 목표

4주 내에는 과도하게 많은 외부 데이터를 붙이기보다, 실질적으로 다음주 예측에 중요한 변수부터 단계적으로 검증한다.

이번 MVP의 목표 범위는 `v0 ~ v2`까지로 설정한다.

 '''text
 v0. 내부 판매/품절 데이터 기반 baseline
 v1. 캘린더/특일 + 날씨 데이터 반영
 v2. 유동성 + 상권 + 연령대 + 추정 소비 정보 반영
 '''

### 1.2 이번 MVP에서 제외하는 항목

아래 항목은 유용할 수 있으나, 4주 내 MVP 범위에서는 제외하고 후속 고도화 후보로 남긴다.

 '''text
 - 학교 학사일정
 - 대기질/미세먼지
 - 지역 행사/축제
 - 원재료 가격
 '''

제외 이유는 다음과 같다.

 '''text
 - 매장 입지에 따라 효과가 크게 달라짐
 - 매장 주변 학교, 측정소, 행사장 등을 매핑하는 추가 작업이 필요함
 - v0~v2가 안정화되기 전에는 추가 변수의 효과를 해석하기 어려움
 - 4주 내에는 핵심 변수의 품질과 예측 성능 검증이 더 중요함
 '''

## 2. 외부 데이터 수집 계획

이번 분석에서 우선적으로 고려할 외부 데이터는 다음 6개 축이다.

| 구분 | 데이터 | 주요 목적 | 데이터 빈도 | 다음주 예측 활용성 |
|---|---|---|---|---|
| 1 | 특일/공휴일 | 휴일, 연휴, 명절, 기념일 효과 반영 | 일 단위 | 매우 높음 |
| 2 | 날씨 | 기온, 강수, 습도, 적설, 일조 영향 반영 | 시간/일 단위 | 높음 |
| 3 | 지하철 승하차 | 매장 주변 유동성 proxy | 시간/일 단위 | 중간~높음 |
| 4 | 경쟁점/상권 점포 | 주변 제과점/카페/편의점 변화 반영 | 점포 단위 | 중간 |
| 5 | 연령대 분포 | 행정동 고객층/가족 수요 proxy | 월 단위 | 중간 |
| 6 | 추정 소비 정보 | 행정동 소비성향 및 구매력 proxy | 분기/연 단위 | 중간 |

## 2.1 특일/공휴일 데이터

### 데이터 출처

한국천문연구원 특일 정보 API  
- 공공데이터포털: https://www.data.go.kr/data/15012690/openapi.do

### 활용 목적

공휴일, 대체공휴일, 명절, 연휴, 기념일 등은 베이커리 판매량에 직접적인 영향을 줄 수 있다.

특히 다음주 예측 관점에서 특일 데이터는 미래값이 확정되어 있으므로, 가장 안정적으로 사용할 수 있는 외부 변수다.

### 주요 파생 변수

 '''text
 - 공휴일 여부
 - 대체공휴일 여부
 - 명절 여부
 - 공휴일 전날 여부
 - 공휴일 다음날 여부
 - 연휴 길이
 - 연휴 n일차
 - 주말 여부
 - 월초/월말 여부
 - 크리스마스, 밸런타인데이, 화이트데이, 어린이날 등 이벤트성 날짜
 - 24절기 여부
 '''

### DB 적재 예시

 '''sql
 external_calendar_daily
 - date
 - is_holiday
 - holiday_name
 - is_public_holiday
 - is_substitute_holiday
 - is_day_before_holiday
 - is_day_after_holiday
 - holiday_streak_length
 - holiday_position_in_streak
 - special_day_type
 '''

## 2.2 날씨 데이터

### 데이터 출처

기상청 ASOS 관측자료  
- ASOS 시간자료: https://www.data.go.kr/data/15057210/openapi.do
- ASOS 일자료: https://www.data.go.kr/data/15059093/openapi.do

### 활용 목적

날씨는 베이커리 수요에 직접적인 영향을 줄 수 있다.

예를 들어 비가 오거나 눈이 오는 날은 방문 수요가 감소할 수 있고, 기온이 낮거나 높은 날에는 특정 품목의 수요가 달라질 수 있다.

### 주요 파생 변수

 '''text
 - 평균 기온
 - 최고 기온
 - 최저 기온
 - 일교차
 - 강수 여부
 - 강수량
 - 강수 지속시간
 - 습도
 - 풍속
 - 적설 여부
 - 적설량
 - 일조시간
 - 체감 날씨 카테고리
 '''

### DB 적재 예시

 '''sql
 external_weather_daily
 - date
 - station_id
 - avg_temp
 - min_temp
 - max_temp
 - precipitation
 - is_rain
 - humidity
 - wind_speed
 - snow_depth
 - sunshine_hours
 - diurnal_range
 '''

### 주의사항

학습 시에는 과거 관측값을 사용하고, 다음주 예측 시에는 기상 예보값을 사용해야 한다.

따라서 운영 단계에서는 관측값과 예보값을 분리하는 것이 좋다.

 '''text
 학습용:
 - weather_observed

예측용:
 - weather_forecast
 '''

## 2.3 지하철 승하차 데이터

### 데이터 출처

서울시 지하철 호선별 역별 시간대별 승하차 인원 정보  
- 서울 열린데이터광장: https://data.seoul.go.kr/dataList/OA-12252/S/1/datasetView.do

### 활용 목적

매장이 역세권에 있거나 유동인구 영향을 많이 받는 입지라면, 지하철 승하차 인원은 매우 유용한 유동성 proxy가 될 수 있다.

### 주요 파생 변수

 '''text
 - 가장 가까운 지하철역 일별 승차 인원
 - 가장 가까운 지하철역 일별 하차 인원
 - 출근 시간대 하차 인원
 - 점심 시간대 승하차 인원
 - 퇴근 시간대 승하차 인원
 - 주말/평일 유동 패턴
 - 최근 4주 동일요일 평균
 - 전주 동일요일 대비 증감률
 - 전년 동주 동일요일 대비 증감률
 '''

### DB 적재 예시

 '''sql
 external_subway_hourly
 - date
 - station_name
 - line_name
 - hour
 - ride_count
 - alight_count

 store_subway_mapping
 - store_id
 - station_name
 - distance_m
 - weight
 '''

### 주의사항

다음주 실제 승하차 인원은 미리 알 수 없으므로, 예측 시에는 실제값 대신 proxy를 사용해야 한다.

 '''text
 다음주 예측 시 사용 가능한 proxy:
 - 최근 4주 동일요일 평균
 - 전년 동주 동일요일 값
 - 공휴일/주말 여부를 반영한 평균값
 - 날씨 보정 유동성 proxy
 '''

## 2.4 경쟁점/상권 점포 데이터

### 데이터 출처

행정안전부 식품 제과점영업 인허가 데이터  
- Open API: https://www.data.go.kr/data/15155252/openapi.do
- File Data: https://www.data.go.kr/data/15044973/fileData.do

필요 시 함께 고려할 수 있는 업종:
- 제과점영업
- 휴게음식점
- 일반음식점
- 커피전문점/카페
- 편의점

### 활용 목적

주변 경쟁점 수와 경쟁점의 개폐업 변화는 장기적인 매출 수준과 수요 분산에 영향을 줄 수 있다.

단순히 현재 점포 수를 사용하는 것이 아니라, 인허가일자와 폐업일자를 기준으로 특정 날짜에 영업 중이었던 점포 수를 재구성하는 것이 중요하다.

### 주요 파생 변수

 '''text
 - 반경 300m 내 제과점 수
 - 반경 500m 내 제과점 수
 - 반경 1km 내 제과점 수
 - 반경 300m/500m/1km 내 카페 수
 - 반경 300m/500m/1km 내 편의점 수
 - 최근 90일 신규 경쟁점 수
 - 최근 180일 신규 경쟁점 수
 - 최근 90일 폐업 경쟁점 수
 - 최근 180일 폐업 경쟁점 수
 - 특정 기준일의 영업 중 경쟁점 수
 '''

### DB 적재 예시

 '''sql
 external_store_license
 - business_id
 - business_name
 - category
 - license_date
 - close_date
 - business_status
 - address
 - lat
 - lon

 store_competitor_daily
 - store_id
 - date
 - competitor_count_300m
 - competitor_count_500m
 - competitor_count_1km
 - cafe_count_500m
 - convenience_store_count_500m
 - new_competitor_90d
 - closed_competitor_90d
 '''

### 주의사항

상권 데이터는 시점 정합성이 중요하다.

 '''text
 특정일 기준 영업 중 점포 조건:
 - license_date <= 기준일
 - close_date is null 또는 close_date > 기준일
 - business_status가 영업/정상 상태
 '''

## 2.5 연령대 분포 데이터

### 데이터 출처

행정안전부 행정동별 성/연령별 주민등록 인구수 API  
- 공공데이터포털: https://www.data.go.kr/data/15108072/openapi.do

### 활용 목적

연령대 분포는 매장의 기본 고객층과 장기적인 수요 baseline을 설명하는 변수로 활용할 수 있다.

베이커리의 경우 영유아/초등 자녀가 있는 가족 비중, 20~40대 비중, 고령층 비중 등이 품목별 수요와 관련될 수 있다.

### 주요 파생 변수

 '''text
 - 행정동 총인구수
 - 0~9세 인구 비중
 - 10대 인구 비중
 - 20~39세 인구 비중
 - 30~49세 여성 비중
 - 60세 이상 인구 비중
 - 가족 수요 proxy
 - 최근 1년 인구 증가율
 - 최근 3년 인구 증가율
 - 영유아/초등 연령대 증가율
 '''

### DB 적재 예시

 '''sql
 external_population_monthly
 - ym
 - admin_dong_code
 - admin_dong_name
 - total_population
 - male_population
 - female_population
 - age_0_9
 - age_10_19
 - age_20_29
 - age_30_39
 - age_40_49
 - age_50_59
 - age_60_plus
 '''

### 주의사항

연령대 분포는 일별 변동을 설명하는 변수라기보다는 매장의 기본 수요 수준과 장기 추세를 보정하는 변수로 보는 것이 적절하다.

## 2.6 추정 소비 정보

서울시 상권분석서비스 소비-행정동

### 활용 목적

소득 수준보다 실제 소비 정보를 보는 것이 더 적절할 수 있다.

행정동별 추정 소비 정보는 해당 지역의 구매력과 소비성향을 나타내는 proxy로 활용할 수 있다.

### 주요 파생 변수

 '''text
 - 행정동 총 소비 규모
 - 음식/소매 관련 소비 규모
 - 연령대별 소비 비중
 - 성별 소비 비중
 - 시간대별 소비 패턴
 - 요일별 소비 패턴
 - 전년 대비 소비 증가율
 '''

### DB 적재 예시

 '''sql
 external_consumption_admin_dong
 - year_quarter
 - admin_dong_code
 - admin_dong_name
 - total_consumption
 - food_consumption
 - retail_consumption
 - age_group_consumption
 - gender_consumption
 - weekday_consumption
 - weekend_consumption
 '''

### 주의사항

서울시 상권분석 소비 데이터는 고빈도 일별 데이터가 아니므로, 일별 수요 변동보다는 상권의 소비 baseline을 설명하는 변수로 활용하는 것이 적절하다.

외부 데이터를 붙이기 전에, 내부 판매량/품절 데이터만으로 어느 정도 예측 가능한지 확인한다.

### 입력 데이터

 '''text
 - 일자
 - 매장 ID
 - 품목 ID
 - 판매량
 - 생산량 또는 입고량

 - 품절 시각
 - 판매 가능 시간
 - 가격
 - 프로모션 여부
 - 휴무 여부
 '''

### 주요 Feature

 '''text
 - 요일
 - 주말 여부
 - 전일 판매량
 - 전주 동일요일 판매량
 - 최근 7일 이동평균

 - 최근 4주 동일요일 평균

 '''text
 - 판매량이 실제 수요를 대표하는지 확인

 - 품목별 baseline 예측 성능

## 3.2 v1. 캘린더 + 날씨 반영 모델

### 목적

다음주 예측에 가장 직접적으로 사용할 수 있는 외부 변수인 특일/공휴일과 날씨를 반영한다.

### 추가 데이터

 '''text
 - 특일/공휴일 데이터
 - 기상 관측 데이터
 - 기상 예보 데이터
 '''

### 주요 Feature

 '''text
 - 공휴일 여부
 - 공휴일 전날/다음날 여부
 - 명절 여부

## 6. 품절 데이터 처리 방향

베이커리 판매량 데이터에서 품절은 매우 중요한 이슈다.

품절이 발생한 날의 판매량은 실제 수요가 아니라, 재고가 있었을 때 관측 가능한 최대 판매량에 의해 잘린 값일 수 있다.

예를 들어 어떤 품목이 오후 2시에 품절되어 80개 판매되었다면, 실제 잠재 수요는 80개보다 높았을 가능성이 있다.

### 처리 방향

 '''text
 1. 품절이 없는 날 중심으로 기본 수요 패턴 학습
 2. 품절 발생일은 censored demand로 별도 표시
 3. 품절 시각이 빠를수록 실제 수요가 더 높았을 가능성을 반영
 4. 판매량 예측과 별도로 품절 위험 예측 모델 검토
 '''

### 추가로 필요한 내부 데이터

 '''text
 - 품목별 생산량 또는 입고량
 - 품목별 폐기량
 - 품목별 품절 여부
 - 품목별 품절 시각
 - 영업시간
 - 중간 추가 생산 또는 추가 입고 여부
 - 가격 변경 이력
 - 프로모션/할인 여부
 '''

## 7. 복수 매장 데이터 요청

## 7.1 요청 배경

현재 1개 매장 데이터만으로도 초기 분석은 가능하다.

다만 이번 분석의 목적이 단일 매장 최적화에 그치지 않고, 외부 데이터가 실제로 수요예측 성능에 기여하는지 검증하는 것이라면 복수 매장 데이터가 필요하다.

1개 매장만으로는 다음을 구분하기 어렵다.

 '''text
 - 모델이 일반적인 수요 패턴을 학습한 것인지
 - 해당 매장의 반복 패턴만 외운 것인지
 - 날씨/공휴일/지하철/상권/연령대/소비정보가 실제로 효과가 있는지
 - 특정 매장 운영 이슈가 모델 성능에 영향을 준 것인지
 - 향후 다른 매장으로 확장 가능한 모델인지
 '''

## 7.2 요청 매장 수

현실적인 요청 수준은 다음과 같다.

 '''text
 최소 요청:
 - 기존 대표 매장 1곳 + 추가 2곳
 - 총 3개 매장

 권장 요청:
 - 상권 유형이 다른 3~5개 매장

 이상적 요청:
 - 안정 매장, 역세권, 주거지, 오피스, 학교/학원가 등 5개 이상
 '''

## 7.3 요청하고 싶은 매장 유형

 '''text
 - 안정적으로 매출이 나오는 대표 매장
 - 역세권 또는 유동인구 영향이 큰 매장
 - 주거지/학교/학원가 인근 매장
 - 오피스 상권 매장
 - 변동성이 큰 매장
 '''

## 7.4 복수 매장 데이터가 필요한 이유

복수 매장을 확보하면 다음 분석이 가능하다.

 '''text
 - 공통 수요 패턴과 매장별 특수성 분리
 - 상권 유형별 외부 변수 효과 비교
 - 역세권 매장에서 지하철 변수가 유효한지 검증
 - 주거지/학교 인근 매장에서 연령대/소비정보가 유효한지 검증
 - 오피스 상권에서 평일/점심/퇴근 수요 패턴 검증
 - 향후 다른 매장으로 확장 가능한 모델인지 판단
 '''

## 8. 데이터 제공 요청 문안 초안

현재 1개 매장 데이터만으로도 초기 수요예측 PoC는 진행 가능할 것으로 보입니다.

다만 이번 분석의 목적이 단일 매장 최적화에 그치지 않고, 날씨, 공휴일, 지하철 승하차, 상권 점포, 연령대, 소비정보 등 외부 데이터가 실제로 예측 성능에 기여하는지 검증하는 것이라면 가능한 범위 내에서 복수 매장 데이터를 함께 제공받는 것이 좋을 것 같습니다.

1개 매장만으로는 모델이 해당 매장의 반복 패턴을 학습한 것인지, 외부 변수가 실제로 수요 변동을 설명한 것인지 구분하기 어렵습니다.

가능하다면 안정적으로 매출이 발생하는 대표 매장 1곳 외에, 상권 유형이 다른 매장 2~4곳을 추가로 제공받고 싶습니다.

예를 들면 다음과 같은 구성이 가장 좋습니다.

 '''text
 - 안정적으로 매출이 나오는 대표 매장
 - 역세권 또는 유동인구 영향이 큰 매장
 - 주거지/학교/학원가 인근 매장
 - 오피스 상권 매장
 - 변동성이 큰 매장
 '''

이렇게 복수 매장을 확보하면, 매장별 특수성과 공통 패턴을 분리할 수 있고, 향후 다른 매장으로 확장 가능한 수요예측 모델인지도 더 현실적으로 검증할 수 있습니다.
